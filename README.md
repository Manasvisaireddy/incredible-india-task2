## вљ™пёЏ How Nunjucks + Vite Work Together

### The Custom Vite Plugin (in `vite.config.js`)

Vite doesn't support Nunjucks out of the box.  
A **custom inline Vite plugin** is written inside `vite.config.js`:

```js
function vitePluginNunjucks(options = {}) {
  return {
    name: 'vite-plugin-nunjucks',
    transformIndexHtml: {
      order: 'pre',          // runs BEFORE Vite's own HTML pipeline
      handler(html) {
        const env = nunjucks.configure(templatesDir, { autoescape: false })
        return env.renderString(html, options.variables)
      }
    }
  }
}
```

**Flow:**
1. Each `.html` entry file contains Nunjucks tags (`{% include %}`, `{% for %}`, `{% set %}`, `{{ variable }}`)
2. On every request (dev) or build, the plugin calls `nunjucks.renderString()` to expand all tags в†’ pure HTML
3. Vite then processes the resulting HTML (inlines assets, hashes filenames, etc.)
4. Output lands in `dist/`

### Data в†’ Templates

All shared data (36 states, 3 packages) lives in `src/data/siteData.js`.  
`vite.config.js` imports this and passes it as `variables` to the plugin:

```js
import { states, packages } from './src/data/siteData.js'

vitePluginNunjucks({ variables: { states, packages } })
```

Inside templates the data is used as:

```html
{% for state in states %}
  <div data-region="{{ state.region }}">{{ state.name }}</div>
{% endfor %}
```
