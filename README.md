## ⚙️ How Nunjucks + Vite Work Together

### The Custom Vite Plugin (in `vite.config.js`)

Vite doesn't support Nunjucks out of the box.  
A **custom inline Vite plugin** is written inside `vite.config.js`:

```js
function vitePluginNunjucks(options = {}) {
  return {
    name: 'vite-plugin-nunjucks',
    transformIndexHtml: {
      order: 'pre',          // runs BEFORE Vite's own HTML pipeline
      handler(html) {
        const env = nunjucks.configure(templatesDir, { autoescape: false })
        return env.renderString(html, options.variables)
      }
    }
  }
}
```

**Flow:**
1. Each `.html` entry file contains Nunjucks tags (`{% include %}`, `{% for %}`, `{% set %}`, `{{ variable }}`)
2. On every request (dev) or build, the plugin calls `nunjucks.renderString()` to expand all tags → pure HTML
3. Vite then processes the resulting HTML (inlines assets, hashes filenames, etc.)
4. Output lands in `dist/`

### Data → Templates

All shared data (36 states, 3 packages) lives in `src/data/siteData.js`.  
`vite.config.js` imports this and passes it as `variables` to the plugin:

```js
import { states, packages } from './src/data/siteData.js'

vitePluginNunjucks({ variables: { states, packages } })
```

Inside templates the data is used as:

```html
{% for state in states %}
  <div data-region="{{ state.region }}">{{ state.name }}</div>
{% endfor %}
```

## 🚀 Installation & Running

### Prerequisites
- **Node.js** ≥ 18  
- **npm** ≥ 9

### 1 — Clone & Install

```bash
git clone https://github.com/YOUR_USERNAME/incredible-india-task2.git
cd incredible-india-task2
npm install
```

### 2 — Add the Logo Image

Copy `logo.png` from the Task 1 repo into:

```
src/assets/images/logo.png
```

*(or any square PNG — the site uses it in the navbar & footer)*

### 3 — Development Server

```bash
npm run dev
```

Vite starts a dev server at `http://localhost:5173`.  
Nunjucks templates are rendered on every request; hot-reload works on CSS/JS changes.

### 4 — Production Build

```bash
npm run build
```

Compiled, minified output is written to `dist/`.

### 5 — Preview Production Build

```bash
npm run preview
```

Serves the `dist/` folder locally at `http://localhost:4173`.

## 📌 Nunjucks Features Used

| Feature | Where Used |
|---------|-----------|
| `{% include "partials/head.njk" %}` | All 3 pages — shared `<head>` |
| `{% include "partials/navbar.njk" %}` | All 3 pages — sticky navbar |
| `{% include "partials/footer.njk" %}` | All 3 pages — footer + scripts |
| `{% include "partials/modal.njk" %}` | `index.html` — package modal |
| `{% set pageTitle = "..." %}` | All 3 pages — dynamic `<title>` |
| `{% set activePage = "..." %}` | All 3 pages — navbar active link |
| `{% for state in states %}` | `index.html` — renders all 36 state cards |
| `{% for pkg in packages %}` | `index.html` — renders 3 package cards |
| `{% set faiths = [...] %}` + `{% for %}` | `about.html` — 10 faith cards |
| `{% set traditions = [...] %}` + `{% for %}` | `about.html` — 5 tradition cards |
| `{% set services = [...] %}` + `{% for %}` | `about.html` — 4 service cards |
| `{% set testimonials = [...] %}` + `{% for %}` | `about.html` — 3 testimonial cards |
| `{% set faqs = [...] %}` + `{% for %}` | `contact.html` — FAQ accordion |
| `{% set regions = [...] %}` + `{% for %}` | `contact.html` — region `<select>` |
| `{{ loop.index }}` | `contact.html` — unique accordion IDs |
| `{{ 'active' if activePage == 'home' }}` | `navbar.njk` — conditional class |
